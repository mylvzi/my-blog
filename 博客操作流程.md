# 博客操作流程

这份文档用于记录当前 Hexo 博客的日常维护流程，包括发布新博客、修改博客、删除博客、预览、部署和常见问题处理。

当前博客源码目录：

```powershell
D:\myblog\my-blog
```

核心目录和文件：

| 路径 | 用途 |
| --- | --- |
| `source\_posts` | 博客文章 Markdown 文件目录 |
| `source\images\posts` | 博客文章图片目录 |
| `deploy.bat` | 构建、提交、推送源码的发布脚本 |
| `tools\publish-obsidian-post.ps1` | 从本地 Markdown/Obsidian 笔记发布文章的辅助脚本 |
| `tools\delete-blog-post.ps1` | 删除文章及关联图片的辅助脚本 |
| `_config.yml` | Hexo 主配置 |
| `_config.stellar.yml` | Stellar 主题配置 |

## 1. 总体发布机制

这个博客是 Hexo 项目，源码推送到远程仓库 `origin/main` 后，由 GitHub Actions 自动部署到 `mylvzi.github.io`。

日常操作可以理解为：

```text
本地写文章
  -> 放入 source\_posts
  -> 本地构建检查
  -> 提交并推送 main 分支
  -> GitHub Actions 自动发布到线上站点
```

`deploy.bat` 已经把「构建、提交、推送」合在一起了。正常情况下，不需要手动执行 `git add`、`git commit`、`git push`，直接运行 `deploy.bat` 即可。

## 2. 发布新博客

### 2.1 适用场景

你已经在本地写好了一篇 Markdown 文件，例如：

```powershell
C:\Users\绿字\Desktop\我的新文章.md
```

希望把它发布到博客线上站点。

### 2.2 推荐方式：使用发布脚本

进入博客目录：

```powershell
cd /d D:\myblog\my-blog
```

执行发布命令：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\publish-obsidian-post.ps1 "C:\Users\绿字\Desktop\我的新文章.md"
```

脚本会自动做这些事情：

1. 把本地 Markdown 文件复制到 `source\_posts`。
2. 自动补全或重写文章 front matter。
3. 处理 Obsidian 图片语法和本地图片引用。
4. 执行 `npm run build` 检查 Hexo 是否能正常生成。
5. 默认执行 `deploy.bat`，把源码推送到远程仓库。
6. 远程仓库收到推送后，由 GitHub Actions 自动部署线上站点。

### 2.3 指定分类、标签和 slug

如果文章需要明确分类和标签，使用：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\publish-obsidian-post.ps1 "C:\Users\绿字\Desktop\我的新文章.md" -Category "技术" -Tags "Java,算法,LeetCode" -Slug "java-algorithm-note"
```

参数说明：

| 参数 | 作用 | 示例 |
| --- | --- | --- |
| `-Category` | 文章分类 | `"技术"` |
| `-Tags` | 文章标签，多个标签用英文逗号分隔 | `"Java,算法,LeetCode"` |
| `-Slug` | 图片目录和文章资源标识，建议用英文短横线 | `"java-algorithm-note"` |
| `-NoDeploy` | 只导入和构建，不推送远程 | `-NoDeploy` |
| `-RestartServer` | 发布后重启本地预览服务 | `-RestartServer` |

推荐 slug 命名：

```text
主题英文关键词-二级关键词
```

例如：

```text
binary-search
recursion-intuition
wechat-mp-publisher-skill
```

### 2.4 只导入文章，先不发布

如果你想先检查效果，不马上推到远程：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\publish-obsidian-post.ps1 "C:\Users\绿字\Desktop\我的新文章.md" -NoDeploy
```

然后启动本地预览：

```powershell
npm run start
```

浏览器打开：

```text
http://localhost:4000
```

确认没问题后再发布：

```powershell
.\deploy.bat "发布新文章：我的新文章"
```

### 2.5 手动发布方式

如果不使用脚本，也可以手动操作。

第一步，把 `.md` 文件放到：

```powershell
D:\myblog\my-blog\source\_posts
```

第二步，确保文章顶部有 front matter：

```markdown
---
title: 文章标题
date: 2026-05-22 22:00:00
tags:
  - Java
  - 算法
categories:
  - 技术
comment: true
summary: 这里写一段文章摘要。
---
```

第三步，如果文章有图片，建议放到：

```powershell
D:\myblog\my-blog\source\images\posts\文章slug
```

文章里使用这种路径：

```markdown
![图片说明](/images/posts/文章slug/img_01.png)
```

第四步，构建检查：

```powershell
npm run build
```

第五步，发布：

```powershell
.\deploy.bat "发布新文章：文章标题"
```

## 3. 修改已发布博客

### 3.1 适用场景

文章已经发布到线上，但你想修改错别字、补充内容、替换图片、调整标题、分类或标签。

### 3.2 修改文章正文

找到文章文件：

```powershell
D:\myblog\my-blog\source\_posts
```

直接编辑对应的 `.md` 文件。

修改后先构建检查：

```powershell
cd /d D:\myblog\my-blog
npm run build
```

构建成功后发布：

```powershell
.\deploy.bat "更新文章：文章标题"
```

### 3.3 修改标题、日期、分类、标签

修改文章顶部 front matter，例如：

```markdown
---
title: 新标题
date: 2026-05-22 22:00:00
tags:
  - Java
  - 并发
categories:
  - 技术
comment: true
summary: 新摘要。
---
```

注意：

1. `title` 会影响文章显示标题。
2. `date` 会影响文章排序。
3. `tags` 和 `categories` 会影响标签页、分类页。
4. 冒号后建议保留空格。
5. 中文、特殊符号较多时，标题可以加引号。

### 3.4 修改文章图片

图片一般放在：

```powershell
D:\myblog\my-blog\source\images\posts\文章slug
```

文章中引用：

```markdown
![图片说明](/images/posts/文章slug/img_01.png)
```

替换图片时，优先保持文件名不变，这样文章引用不用改。

如果新增图片，建议按顺序命名：

```text
img_01.png
img_02.png
img_03.png
```

## 4. 删除博客

### 4.1 推荐方式：使用删除脚本

进入博客目录：

```powershell
cd /d D:\myblog\my-blog
```

按标题、文件名或关键词删除：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\delete-blog-post.ps1 "文章标题关键词"
```

脚本会自动做这些事情：

1. 在 `source\_posts` 中查找匹配文章。
2. 如果匹配多篇，会停止并提示你输入更精确的关键词。
3. 找到唯一文章后，显示将删除的文章文件。
4. 自动识别文章里引用的 `/images/posts/xxx/` 图片目录。
5. 删除前要求输入 `DELETE` 确认。
6. 不会直接永久删除，而是移动到 `.trash\deleted-posts\时间戳`。
7. 执行 `npm run build` 检查博客是否还能正常构建。
8. 默认执行 `deploy.bat` 推送远程，让线上站点同步删除。

### 4.2 删除但暂不发布

如果想先本地确认删除效果：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\delete-blog-post.ps1 "文章标题关键词" -NoDeploy
```

确认后再执行：

```powershell
.\deploy.bat "删除文章：文章标题"
```

### 4.3 跳过确认

确认关键词一定只匹配目标文章时，可以加 `-Force`：

```powershell
pwsh -ExecutionPolicy Bypass -File .\tools\delete-blog-post.ps1 "文章标题关键词" -Force
```

谨慎使用 `-Force`。虽然脚本会移动到 `.trash`，但误删后仍然需要手动恢复。

### 4.4 手动删除方式

不推荐，但可以这样做：

1. 删除 `source\_posts` 下对应的 `.md` 文件。
2. 删除 `source\images\posts` 下对应图片目录。
3. 执行构建检查：

```powershell
npm run build
```

4. 发布：

```powershell
.\deploy.bat "删除文章：文章标题"
```

cd /d D:\myblog\my-blog

http://localhost:4000

## 6. 构建和发布命令

### 6.1 安装依赖

第一次使用或 `node_modules` 不存在时：

```powershell
npm install
```

### 6.2 清理并生成静态站点

```powershell
npm run build
```

这个命令会执行：

```text
hexo clean & hexo generate
```

适合在每次发布前检查。

### 6.3 一键发布

```powershell
.\deploy.bat "提交说明"
```

例如：

```powershell
.\deploy.bat "发布新文章：递归入门"
.\deploy.bat "更新文章：二分查找边界"
.\deploy.bat "删除文章：旧文章标题"
```

`deploy.bat` 会：

1. 进入项目目录。
2. 检查 npm 是否可用。
3. 如果没有 `node_modules`，自动执行 `npm install`。
4. 执行 `npm run build`。
5. `git add -A`。
6. 如果有变更，自动提交。
7. 推送到 `origin/main`。
8. 如果直连推送失败，会尝试使用本地代理 `http://127.0.0.1:7897`。
9. 提示 GitHub Actions 会部署到 `mylvzi.github.io`。

## 7. 推荐工作流

### 7.1 发布新文章的标准流程

```text
1. 在本地写好 Markdown。
2. 确认标题、摘要、分类、标签。
3. 使用 publish-obsidian-post.ps1 导入。
4. 本地 npm run build。
5. 可选：npm run start 预览。
6. 运行 deploy.bat 发布。
7. 等 GitHub Actions 完成部署。
8. 打开线上博客确认文章已出现。
```

推荐命令：

```powershell
cd /d D:\myblog\my-blog
pwsh -ExecutionPolicy Bypass -File .\tools\publish-obsidian-post.ps1 "C:\Users\绿字\Desktop\我的新文章.md" -Category "技术" -Tags "Java,算法" -Slug "my-new-post"
```

### 7.2 修改文章的标准流程

```text
1. 打开 source\_posts 中的文章。
2. 修改正文、front matter 或图片引用。
3. 执行 npm run build。
4. 本地预览确认。
5. 执行 deploy.bat 发布更新。
```

推荐命令：

```powershell
cd /d D:\myblog\my-blog
npm run build
.\deploy.bat "更新文章：文章标题"
```

### 7.3 删除文章的标准流程

```text
1. 用标题或文件名关键词定位文章。
2. 执行 delete-blog-post.ps1。
3. 输入 DELETE 确认。
4. 脚本自动移动文章和图片到 .trash。
5. 脚本自动构建并发布。
6. 线上确认文章已消失。
```

推荐命令：

```powershell
cd /d D:\myblog\my-blog
pwsh -ExecutionPolicy Bypass -File .\tools\delete-blog-post.ps1 "文章标题关键词"
```

## 8. 常见问题处理

### 8.1 构建失败

先执行：

```powershell
npm run build
```

重点检查：

1. front matter 是否有格式错误。
2. `title:`、`date:`、`tags:`、`categories:` 的缩进是否正确。
3. Markdown 中代码块是否正确闭合。
4. 图片路径是否写错。
5. 文件名是否包含异常符号。

常见 YAML 正确写法：

```markdown
---
title: "包含冒号：或特殊符号的标题"
date: 2026-05-22 22:00:00
tags:
  - Java
  - 算法
categories:
  - 技术
comment: true
summary: "摘要内容"
---
```

### 8.2 图片不显示

检查文章中的图片路径是否以 `/images/posts/` 开头：

```markdown
![图片说明](/images/posts/文章slug/img_01.png)
```

再检查文件是否存在：

```powershell
Test-Path "D:\myblog\my-blog\source\images\posts\文章slug\img_01.png"
```

不要使用本机绝对路径：

```markdown
![错误示例](C:\Users\绿字\Desktop\img_01.png)
```

线上站点无法读取你电脑里的本地路径。

### 8.3 推送失败

`deploy.bat` 会先直接 `git push origin main`，失败后会检查本地代理 `127.0.0.1:7897` 并重试。

如果仍然失败，检查：

1. 网络是否正常。
2. GitHub 是否能访问。
3. 本地代理是否启动。
4. 当前分支是否是 `main`。
5. 远程仓库权限是否正常。

可手动查看状态：

```powershell
git status
git remote -v
git branch
```

### 8.4 GitHub Actions 部署后线上没更新

可能原因：

1. GitHub Actions 还没跑完。
2. 浏览器缓存。
3. 构建成功但文章日期太旧，排序不在首页。
4. 文章 front matter 中 date 写错。
5. 推送到了错误分支。

处理方式：

```powershell
git log --oneline -5
git status
```

然后去 GitHub 仓库的 Actions 页面确认最新 workflow 是否成功。

### 8.5 删除后想恢复

删除脚本不会直接永久删除，会把文件移动到：

```powershell
D:\myblog\my-blog\.trash\deleted-posts\时间戳
```

恢复步骤：

1. 找到对应时间戳目录。
2. 把 `.md` 文件移回 `source\_posts`。
3. 把图片目录移回 `source\images\posts`。
4. 执行：

```powershell
npm run build
.\deploy.bat "恢复文章：文章标题"
```

## 9. 操作前检查清单

发布前建议快速检查：

- 文章文件在 `source\_posts`。
- front matter 存在且 YAML 格式正确。
- 标题、日期、分类、标签符合预期。
- 图片路径不是本机绝对路径。
- `npm run build` 成功。
- 本地预览页面没有明显排版问题。
- 发布时提交说明清楚，例如 `发布新文章：xxx`、`更新文章：xxx`、`删除文章：xxx`。

删除前建议快速检查：

- 查询关键词只匹配一篇目标文章。
- 确认关联图片目录确实属于这篇文章。
- 如不确定，先加 `-NoDeploy` 本地检查。
- 确认后再发布到远程。
